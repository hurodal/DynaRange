 Trabajar con GIT.
 Creación de ramas nuevas y mezcla de código final. Pasos.

1 Crear y cambiar a una nueva rama
	(Para empezar a trabajar en una nueva funcionalidad o bug)

	git checkout -b nombre-de-la-rama


2 Trabajar y hacer commits locales

	(Modificas, guardas, y confirmas los cambios en tu rama)

	git add .
	git commit -m "Mensaje descriptivo del cambio"

	(Repites este paso las veces que necesites)


3 Preparar la fusión (merge)

	(Cuando has terminado el trabajo en tu rama)
    Vuelves a la rama principal:
    
    git checkout main

    Actualizas la rama principal con los cambios del remoto por si 
    alguien ha modificado algo mientra tú trabajas en la nueva rama:
    
    git pull origin main


4 Fusionar los cambios

	(Integras el trabajo de tu rama en la rama principal)

	git merge nombre-de-la-rama


5 Subir el resultado al remoto

	(Publicas los cambios ya fusionados para que otros los vean)
	git push origin main


6 Limpiar el entorno

	(Una vez fusionada, la rama de trabajo ya no es necesaria)
    Borrar la rama local:
    
    git branch -d nombre-de-la-rama

    Borrar la rama remota (si la subiste alguna vez):

    git push origin --delete nombre-de-la-rama
